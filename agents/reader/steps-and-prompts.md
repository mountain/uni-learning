我建议定为**正式四遍**，外加一个**零号预处理**。
这样最稳，也最适合交给 agent 分工执行。

# 总体结构

**零号预处理**：编目与分流
**第一遍**：原初还原
**第二遍**：结构抽取
**第三遍**：论证拓扑
**第四遍**：回顾—批判—映射

然后每读完 **5 篇**，再做一次**组综合**。
这样不会淹没在材料里。

---

# 零号预处理：编目与分流

这一遍不算正式阅读，只做入口整理。

## 目标

把文献放进正确的问题域，防止后面读乱。

## agent 输入上下文

只给：

* 题目
* 作者
* 年份
* 摘要
* 目录
* 若有前言/引言，也可给前 1–2 页

不要给次级评论，不要给现代解释。

## 输出要求

输出一张很短的编目卡：

1. 文献类型：技术论文 / 立场文本 / 会议 proposal / 综述 / 学科史
2. 主路线归属：

   * cybernetics / automata / Dartmouth
   * Bayes / probability
   * Solomonoff / universal induction
   * open computation / limit / anytime
   * Transformer / ICL / PFN
   * LM / agent / AGI
3. 预估难点：概念密度 / 数学密度 / 历史背景密度
4. 推荐后续阅读优先级
5. 推荐服务于 survey 的哪一章

## 零号 prompt 的核心

> 你不是在总结内容，而是在做入口编目。请根据题目、摘要、目录与引言，判断这篇文献属于哪条路线、将来最可能服务于 survey 的哪一部分，并指出它在阅读中最需要警惕的误读风险。

---

# 第一遍：原初还原

这是最重要的一遍。
目的不是评价对错，而是进入作者当时的视野。

## 目标

重建作者所面对的**原始问题场**与**心理地图**。

## agent 输入上下文

只给：

* 原文全文
* 零号编目卡

**不要给**现代综述，不要给我们自己的大地图解释，不要给后见之明材料。
这一遍要尽量隔离“现代术语污染”。

## 输出要求

输出“原初还原卡”，固定 8 项：

1. 这篇文章在回应什么原始问题？
2. 作者认为这个问题为什么重要？
3. 作者脑中那张更大的地图是什么？
4. 作者把哪些对象看成同一整体？
5. 核心概念 3–7 个
6. 方法承诺是什么？
7. 作者主动切掉了哪些边界？
8. 一句压缩：这篇文章最像是在开辟什么问题空间？

## 第一遍的约束

* 禁止用“后来证明”“今天看来”这类句式
* 禁止直接把旧文献翻译成今天学科标签
* 允许引用作者的愿景句、排除句、定义句

## 第一遍 prompt 的核心

> 请把这篇文献读成一个尚未分裂的问题场。你的任务不是判断其正确性，而是重建作者当时以为自己在探索什么。请避免使用后来的学科标签覆盖原文，优先抽取作者自己的对象边界、问题压力和心理地图。

---

# 第二遍：结构抽取

这一遍开始把原文拆成结构。

## 目标

抽出文献内部的**概念骨架**与**结构骨架**。

## agent 输入上下文

给：

* 原文全文
* 零号编目卡
* 第一遍“原初还原卡”

仍然**不要给**现代评论和我们自己的批判框架。

## 输出要求

输出“结构抽取卡”，固定 7 项：

1. 核心论题
2. 核心概念表
3. 核心二分或张力
4. 核心类比
5. 核心假设
6. 核心限制
7. 核心未解问题

然后补一张“关键句表”：

* 定义句
* 排除句
* 愿景句
* 转折句

再补一句“结构骨架”：

> 本文通过什么结构把问题组织起来。

## 第二遍的约束

* 还不做强批评
* 还不和今天路线硬对应
* 重点是把原文里的骨架抽干净

## 第二遍 prompt 的核心

> 请在不引入外部解释的前提下，把这篇文献的内部结构抽取出来。重点识别其核心概念、二分、类比、限制与未解问题，并用最少的结构语句概括本文如何组织论证对象。

---

# 第三遍：论证拓扑

这是你刚才特别强调的新环节。
也是把“细节的魔鬼”还原出来的关键。

## 目标

重建作者从起点走到结论的**大步骤、依赖、意义、脆弱点**。

## agent 输入上下文

给：

* 原文全文
* 零号编目卡
* 第一遍输出
* 第二遍输出

必要时可再给：

* 附录
* 文中的关键引理或公式部分

这一遍依然**尽量不引入外部现代评论**，因为要先把作者自己的论证坡道重建出来。

## 输出要求

输出两张核心表。

### 表 A：论证路线分解表

每篇拆成 **5–9 个大步骤**：

| 步骤 | 该步主张 | 直接依赖 | 该步作用 | 若移除此步会怎样 |

### 表 B：依赖—意义分析表

| 步骤 | 依赖类型 | 隐含假设 | 局部意义 | 全局意义 |

依赖类型强制分成五类：

* 概念依赖
* 形式依赖
* 经验依赖
* 方法依赖
* 制度性依赖

然后再补一张“脆弱点扫描表”：

* 跃迁点
* 沉默点
* 排他点

最后补一句总判断：

> 这篇文章真正重要的，不只是结论，而是哪条论证路线。

## 第三遍的约束

* 必须按“推进台阶”拆，不按段落拆
* 每一步都回答：如果没有这一步，后面哪句话说不出来
* 要区分局部意义和全局意义

## 第三遍 prompt 的核心

> 请把这篇文献当作一条论证路线，而不是一个结论容器。你的任务是拆出作者从问题起点走到结论的关键台阶，分析每一步的依赖与作用，并标出这条路线最脆弱、最有方法论意味的地方。

---

# 第四遍：回顾—批判—映射

前三遍尽量忠于原文。
第四遍才把它放回今天的大地图。

## 目标

把这篇文献变成对 survey 有用的判断。

## agent 输入上下文

给：

* 原文全文
* 前三遍全部输出
* 我们 survey 的总概念地图
* 当前章节提纲
* 少量必要的现代参考点

这里可以开始引入现代语境，但要**限量**。
我建议每篇最多再给 3–5 个现代锚点文献，不要太多。

## 输出要求

输出“回顾—批判卡”，固定 6 项：

1. 命中型判断：哪些洞见后来被证实
2. 误判型判断：哪些地方过窄/过宽
3. 遮蔽型判断：哪些概念后来被拆散、改名或边缘化
4. 回流型判断：哪些支线今天重新重要
5. 与今天路线的对应表
6. 未来生发表

### 对应表

| 历史步骤 | 今天的对应 | 对应强度 | 是否发生语义变形 |

### 未来生发表

| 历史线索 | 今天尚未完全展开的原因 | 将来可能与哪条路线汇合 |

最后再给两句：

* **这篇文献对本 survey 最有价值的贡献是什么？**
* **它应该被写进哪一章，作为哪一种证据？**
  证据类型强制标记为：

  * Theorem
  * Construction
  * Interpretation
  * Historical correction

## 第四遍的约束

* 批判必须针对结构，不要只做对错裁判
* 对应必须说明是“同构 / 类比 / 弱相似”
* 未来生发必须基于前三遍，不可空想

## 第四遍 prompt 的核心

> 现在请把这篇文献放回我们的总概念地图中。你的任务不是重复内容，而是判断它在今天仍然改变了什么、遮蔽了什么、回流了什么，并分析它与当前路线以及未来可能路线之间的结构对应关系。

---

# 为什么是四遍，而不是三遍或五遍

## 不是三遍

因为你现在新增的“论证路线依赖和意义分析”不能夹在普通结构抽取里，否则会被做浅。
它必须独立成一遍。

## 不必五遍

如果再多拆一遍，成本上升很快，收益反而不明显。
四遍已经足够把：

* 原初状态
* 结构骨架
* 论证坡道
* 今日映射

这四层都分开。

所以我建议固定为：

**零号预处理 + 四遍正式阅读 + 五篇一综合**

这是最稳的工作节奏。

---

# 五篇一综合：不是阅读遍数，而是批次整合

每完成 5 篇，做一次组综合。
这一步最好由另一个 agent 或一个“综合 agent”来做。

## 输入上下文

* 5 篇的四遍输出
* 当前 survey 章节提纲

## 输出要求

1. 这 5 篇共同回应什么问题
2. 共享什么心理地图
3. 最大分歧在哪里
4. 哪些概念在这组文献中发生命运变化
5. 对 survey 哪一章最关键
6. 应形成什么图或表

## 组综合 prompt 的核心

> 请不要逐篇复述。请把这 5 篇文献视为一个小地层，分析它们共同构成了什么问题空间、出现了哪些分岔与回流、以及它们如何共同服务于我们的 survey 某一章节。

# agent 分工建议

若你要并行推进，我建议至少分成四类 agent 角色：

### 1. 编目 agent

做零号预处理

### 2. 原初还原 agent

专做第一遍
要求尽量“无后见之明”

### 3. 结构—论证 agent

做第二、三遍
要求逻辑清晰、善于拆步骤

### 4. 批判—映射 agent

做第四遍和五篇综合
要求能连接今天问题与未来路线

这样分工会比一个 agent 从头做到尾更稳，因为不同遍的认知姿态不同。

---

# 上下文控制的一个关键原则

这是很重要的操作原则：

## 第一遍、第二遍、第三遍

尽量**封闭上下文**，避免被现代叙事污染。

## 第四遍和组综合

再**打开上下文**，允许与今天的地图接通。

你可以把它理解成：

* 前三遍是在“文献内部”工作
* 第四遍是在“文献—今天之间”工作

这个边界一定要守住，不然第一遍会被第四遍污染。

---

# 最后压缩成一句工作协议

**每篇历史文献采用“零号编目 + 四遍正式阅读”的流程：第一遍还原问题场，第二遍抽取结构骨架，第三遍重建论证拓扑，第四遍完成回顾批判与今日映射；每五篇再做一次组综合。前三遍尽量封闭上下文，第四遍才接入我们的 survey 总地图。**

如果你愿意，我下一步可以直接把这套流程写成一个**agent 可执行的标准操作手册**，包括每一遍的 prompt 模板、输出 JSON 字段，以及组综合模板。
