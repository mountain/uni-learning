# Programming a Computer for Playing Chess：第一遍（原初还原）

## 书目信息

- 标题：Programming a Computer for Playing Chess
- 作者：Claude E. Shannon
- 年份：1950
- 发表：Philosophical Magazine, Vol.41, pp.256-275

## 第一遍八项输出

### 1) 原始问题

引言将下棋建模为可计算决策问题，强调资源受限条件下的程序构造。

### 2) 问题重要性

论文提出计算机下棋的程序化框架，讨论搜索深度、评估函数与策略近似，在有限资源下定义可行博弈计算路径。

### 3) 更大心理地图

作者将问题组织为：棋类问题与机器求解设定；搜索策略与分支控制；评估函数与启发式近似；算力约束下的程序设计原则。

### 4) 同一整体中的关键对象

- 棋类问题与机器求解设定
- 搜索策略与分支控制
- 评估函数与启发式近似
- 算力约束下的程序设计原则

### 5) 核心概念（3–7）

1. 棋类问题与机器求解设定
2. 搜索策略与分支控制
3. 评估函数与启发式近似
4. 算力约束下的程序设计原则
5. cybernetics / automata / Dartmouth
6. 问题建模
7. 边界条件

### 6) 方法承诺

作者在 cybernetics / automata / Dartmouth 路线内采用结构化建模与分层推进的方法承诺。

### 7) 主动切除的边界

- 优先处理文献中明确定义的问题对象，不外推到未定义任务
- 区分文献内部可证明主张与外部解释性扩展
- 重点防止误读：将启发式搜索直接等同为现代最优规划或强化学习；忽略其核心问题是资源受限下的近似决策而非完全最优

### 8) 一句压缩：该文献开辟了何种问题空间

该文献开辟了一个围绕“cybernetics / automata / Dartmouth”组织问题并以结构化论证推进结论的问题空间。
